# Velocity Cortex 3.0 - Memory Layer

This directory contains the **L4: Knowledge Layer** - JSON-based symbolic knowledge that drives the neurosymbolic AI system.

## 📁 File Structure

### Core Knowledge Files

| File | Purpose | Update Frequency |
|------|---------|------------------|
| `core_protocols.json` | Universal OS shortcuts (Ctrl+C, Ctrl+S, etc.) | Static (manual updates) |
| `os_manual.json` | Installed applications database | Generated by scanner.py |
| `skills.json` | Learned behaviors with RL confidence scores | Dynamic (RL updates) |
| `app_profiles.json` | Application metadata (UI framework, shortcuts) | Semi-static (manual curation) |

### Runtime Files

| File | Purpose | Update Frequency |
|------|---------|------------------|
| `element_cache.json` | Cached UI element locations | Runtime (auto-invalidates) |
| `execution_history.json` | Performance logs for debugging | Runtime (append-only) |
| `processor_config.json` | Cortex processor settings | Static (config file) |

---

## 🔄 Data Flow

```
1. User Intent → Cortex Processor
2. Processor checks skills.json (L4: Symbolic)
3. If confidence > threshold → Execute
4. If fails → Update RL scores in skills.json
5. Log to execution_history.json
```

---

## 🧠 Reinforcement Learning

The `skills.json` file uses confidence-based RL:

- **Success:** `confidence += 0.1` (up to 1.0)
- **Failure:** `confidence -= 0.2` (down to 0.1)
- **Method Selection:** Always try highest confidence first
- **Fallback:** If confidence < threshold, try next layer

## 📝 Schema Definitions

### skills.json Schema
```json
{
  "skill_id": {
    "app": "Application name",
    "description": "What this skill does",
    "aliases": ["alternative", "trigger", "phrases"],
    "methods": [
      {
        "type": "symbolic | structural | visual",
        "action": "hotkey | click_element | click_coords_vlm",
        "confidence": 0.0-1.0,
        "success_count": 0,
        "fail_count": 0,
        "avg_latency_ms": 0
      }
    ]
  }
}
```

### app_profiles.json Schema
```json
{
  "AppName": {
    "executable": "app.exe",
    "ui_framework": "Win32 | UWP | Electron | Qt",
    "supports_ui_automation": true/false,
    "preferred_method": "symbolic | structural | visual",
    "common_shortcuts": {},
    "notes": "Special handling instructions"
  }
}
```

## 🚀 Usage Example

```python
from brain.cortex import learning

# Load skills
skills = learning.load_skills()

# Execute with RL
result = skills["spotify_play"]["methods"][0]
if result["confidence"] > 0.7:
    execute(result)
    
# Update based on outcome
learning.update_confidence("spotify_play", success=True)
```

## ⚠️ Important Notes

1. **Never delete `core_protocols.json`** - These are hardcoded universal truths
2. **`os_manual.json` is generated** - Run `brain/ganglia/scanner.py` to regenerate
3. **`skills.json` learns over time** - Backup before resetting
4. **Cache expires** - `element_cache.json` auto-invalidates after 5 minutes

---

## 🔧 Maintenance

### Regenerate OS Manual
```bash
python brain/ganglia/scanner.py
# Output: brain/memory/os_manual.json
```

### Reset Learning
```bash
# Backup first!
cp brain/memory/skills.json brain/memory/skills.json.backup

# Reset confidence scores to defaults
python brain/cortex/learning.py --reset
```

### Clear Cache
```bash
# Safe to delete anytime
rm brain/memory/element_cache.json
```
